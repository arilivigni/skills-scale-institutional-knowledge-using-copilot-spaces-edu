# OctoAcme Project Management Docs

## Welcome

Welcome to the OctoAcme Project Management Docs. This folder is the central entry point for the processes, roles, and working agreements that guide how OctoAcme plans, delivers, releases, and improves projects.

## Overview

OctoAcme uses a customer-driven project management approach built around iterative delivery, clear ownership, measurable goals, and continuous improvement. Work moves through five core lifecycle phases—Initiation, Planning, Execution, Release, and Retrospective—so teams can align on outcomes early, deliver in manageable increments, and improve with every cycle. The process emphasizes customer-first delivery, data-informed decisions, and psychological safety so teams can raise risks early, learn quickly, and make better decisions together. Clear role definitions for project managers, product managers, developers, QA partners, and stakeholders support accountability throughout the lifecycle, while quality assurance, risk management, and retrospective follow-through help maintain reliable delivery standards.
